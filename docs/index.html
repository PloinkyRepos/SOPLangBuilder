soplangBuilder Agent Documentation

This repository hosts a single agent: soplangAgent (published in Ploinky as soplangBuilder integration layer).

The agent is designed to be integrated in Ploinky and consumed from Explorer-style UI flows, with current usage focused on the file explorer project in .ploinky/repos/AssistOSExplorer.

At runtime the agent is exposed through MCP, and each MCP tool call is executed through soplangAgent/soplang-tool.sh.

Operational Role

The agent bridges SOPLang workspace APIs to MCP tools declared in soplangAgent/mcp-config.json, then executes those operations through a controlled wrapper path. It loads runtime plugins from soplangAgent/plugins/, registers commands, and makes the resulting contract available to Explorer-facing flows.

The same boundary also includes Achilles skill discovery and command exposure, so UI actions can call a stable MCP surface while SOPLang-specific orchestration remains inside soplangBuilder.

Repository Scope

The repository intentionally keeps one agent implementation. It is not a generic multi-agent monorepo. The goal is to keep SOPLang build orchestration and MCP exposure isolated and predictable for Explorer UI consumption.

Validation rule: run npm test from soplangAgent/ for every code change, together with docs and specs updates.
